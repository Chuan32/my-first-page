📗 第6课：数组进阶 forEach / filter / map 2026-05-24

🔄 forEach — 遍历数组

✓ arr.forEach(item => { })
✓ 也能拿下标：arr.forEach((item, index) => { })
✓ 比 for 循环更简洁，不用写 i 和 length
✓ 不返回新数组，只是操作每个元素

🔍 filter — 过滤数组

✓ let 新数组 = arr.filter(item => 条件)
✓ 符合条件的保留，不符合的去掉
✓ 返回新数组，原数组不变
✓ 例：todos.filter(todo => !todo.done) 筛出未完成

🔄 map — 转换数组

✓ let 新数组 = arr.map(item => 新值)
✓ 每个元素"映射"成新值
✓ 返回新数组，长度不变

📌 三个方法对比

! forEach：遍历用（不产生新数组）
! filter：筛选用（长度可能变短）
! map：转换用（长度不变）
! 三个方法都不会改变原数组

📗 第10课：CSS 定位（position）2026-05-25

📌 四种定位模式

✓ position: relative 相对定位：相对自己原来的位置偏移
✓ position: absolute 绝对定位：相对最近定位父元素"贴"过去
✓ position: fixed 固定定位：相对浏览器窗口固定，滚动不动
✓ top / right / bottom / left 方向属性指定位置
✓ z-index 层叠顺序：数字越大越在上面

📌 关键理解

! relative：保留原来位置（占位），偏移不影响其他元素
! absolute：脱离文档流（不占位），父元素要设 position: relative
! fixed：脱离文档流，相对于浏览器窗口定位
! z-index 只对定位元素（非 static）生效

🗄️ 什么是数据库

✓ 数据库 = 专门存数据的"仓库"，永久保存、有结构、可查询
✓ SQLite = 最轻量的数据库，数据存在 .db 文件里，无需装服务器
✓ SQL = 操作数据库的"指令语言"

🔧 CRUD 四大操作

✓ Create → INSERT INTO 表名 (列) VALUES (?)
✓ Read → SELECT * FROM 表名 WHERE 条件
✓ Update → UPDATE 表名 SET 列=? WHERE 条件
✓ Delete → DELETE FROM 表名 WHERE 条件

📦 better-sqlite3 核心 API

✓ new Database('文件名.db') 连接/创建数据库
✓ db.exec('SQL') 执行建表等不返回数据的 SQL
✓ db.prepare('SQL').run(值) 增/删/改
✓ db.prepare('SQL').all() 查所有行（返回数组）
✓ db.prepare('SQL').get(值) 查单行（返回对象）
✓ ? 占位符 = 安全写法，防止 SQL 注入
✓ INTEGER PRIMARY KEY AUTOINCREMENT 自动增长 ID
✓ TEXT NOT NULL 文字类型，不能为空

🚨 第20课犯过的错误

❌ 问题20：let 重复声明同一个变量
let insertStmt = ... 两次 ❌ → 第二次改成 let insertHobby = ... ✅
同一个作用域内 let 不能声明两次同名变量
❌ 问题21：变量名拼写不一致
let insertHobby = ... 却写 insertHobyy.run() ❌
声明和调用必须完全一样 ✅

📌 常犯错误大全（快速查阅）

1️⃣ 函数调用忘记加 ()
onclick="fn" ❌ → onclick="fn()" ✅
2️⃣ 函数名拼写不一致
HTML 里叫 show()，JS 里写 showMessage() ❌ → 名字必须一样 ✅
3️⃣ 对象调用了数组方法
obj.join() ❌ → .join() 只能用在数组上 ✅
4️⃣ if...else 写在函数外面
条件判断必须包在 function { ... } 里，否则不会按预期执行
5️⃣ 布尔值写成字符串
if (done === "true") ❌ → if (done) ✅
6️⃣ innerText vs innerHTML 混用
如果内容包含 HTML 标签（如 <p>），用 innerHTML
如果只是纯文字，用 innerText（更安全）
7️⃣ 多个定时器同时跑
每次点"开始"都 setInterval ❌ → 先 if (timerId !== null) return ✅
8️⃣ innerHTML += 导致事件丢失
innerHTML += ... 会重绘整个容器，已绑定的事件全丢了 ❌
改用 createElement + appendChild ✅
9️⃣ 变量名拼写错误
masageBoard ❌ → messageBoard ✅
classList.container() ❌ → classList.contains() ✅
🔟 alert 放在 if 条件里
if (alert(...)) ❌ → if (条件) { alert(...) } ✅
1️⃣1️⃣ 创建对象忘了加 {}
res.json(message: "收到！", data: req.body) ❌ → res.json({message: "收到！", data: req.body}) ✅
对象必须用 {} 包起来，就像数组必须用 []
